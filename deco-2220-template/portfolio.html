My Awesome Portfolio!

Background:

Pigeon is an application that helps people borrow books from the library. People can use this app to borrow books from the library. The drone will then deliver books to the user’s location and when the due day come, the pigeon will remind its user and send a drone to pick up and return those books.

Problem:

Even if people order books online, they still need to come to the library to pick up books.

User research:

I did a user test for “pigeon”, I interviewed three people and I went to their home to do the user test. All three interviewers were students from the University of Sydney. They all had the experience of borrowing books from the library, and they were willing to try drone delivery books. Firstly, I let them use our app “pigeon” and then I interviewed them. During the interview, I recorded their views and suggestions on “pigeon”.

Figure 1.1 - User test1 Figure 1.2 - User test2 Figure 1.3 - User test3

User test results:

During the user test, all users said the empty order page cannot back to other pages. On the “list” page, the user cannot select the books they want and it also cannot use “select all” button. One of the visitors tried to return to the previous page on the “order delivered” page because he clicked the wrong button, but the app didn’t respond. After that, the user discovers that many pages cannot go back to the previous page or directly jump to the home page.

Figure 1.4 - wireframe

Recommendations：

During the interview, all three users thought that we should make the return button in the upper left corner, so that it can bring users a better user experience and also can make the app look cleaner and easier. Some users suggested unifying the “borrow button” in the booking interface with other buttons so that the page would look more beautiful and integral. One user thought that the add list feature in the bookinfo page could be placed on the left love button to make the interface cleaner and more organized. They also think we should make an add list page so that users can select the books they want on the list page. Two of them thinks we should cancel the face id function on the “confirmation” page, the face id on the confirmation page is redundant because the face id feature is also required for the book to be delivered by the drone.They also suggested that we should make the message reminder window more perfect. Let all the buttons in the app look unified so that the entire app looks more holistic and concise. After that, we modified these issues after the test.

Finial design

In the final design, we tweaked and modified the previous issues. We fixed all the bug existed in the application and we replaced "add a list" button with "list" and "love" icon. All the users think these subtle changes can make the app more elaborate. Besides face id system, at the moment, we integrated an extra manual recognition to cope with the problem they encountered before. We tested and checked the keys on every application page to make sure that each button worked properly to avoid the occurrence of bug. Then we tweaked the small details of the app, Such as changing the appearance of the "back" icon, changed the location of the notification bar, improved the integrity of the app, and made the app more neat and beautiful.
